**TouchBarCraft** is a lightweight, high‑performance utility written in Swift and SwiftUI that gives you full control over your macOS Touch Bar. It replaces the default system Control Strip with custom, interactive widgets configured from a modern GUI.

## ✨ Features

- **🗂 Supercharged Anki Integration**: Full support for studying flashcards directly on your Touch Bar!
  - Syncs seamlessly with **AnkiConnect** (localhost:8765).
  - Displays dynamic question and answer texts with bold/italic/underline HTML tag rendering.
  - Supports **Furigana** (ruby text) parsing and optional combination of furigana into the main line.
  - New **Touch Bar Layout Toggle** – move the media controls (Rating, Audio, Reveal) to the left side, keeping Sync on the right.
  - Configurable **Audio Field** for Touch Bar playback (`touchBarAudioField`).
  - Real‑time remaining card counts (New / Learning / Review) with customizable visibility.
- **Global Touch Bar Override**: Fully replaces the native macOS Touch Bar system‑wide.
- **System Tray Menu Bar Item**: Quick access to controls and settings right from your macOS Menu Bar.
- **Launch at Login (Autostart)**: Uses Apple's modern `SMAppService` API to launch at startup seamlessly.
- **Custom JSON Configuration**: Widgets are dynamically saved and loaded from `~/.touchbarcraft.json`.

## 🗂 Anki Integration Guide

The standout feature of TouchBarCraft is its powerful, native **Anki Flashcard integration** built directly into your Touch Bar. Review cards passively or actively throughout your day without distracting you from your work.

### 🌟 Key Anki Features

- **Live Connection Status**: Automatically detects if Anki is open and connected. Shows a quick "Connect" option on the Touch Bar when offline.
- **HTML Styling Support**: Richly parses `<b>/<strong>`, `<i>/<em>`, and `<u>` tags. Bold text uses a customizable color (`ankiBoldColorHex`).
- **Furigana Support**: Text like `私[わたし]` is rendered as ruby (furigana) above the kanji. You can enable **Combine Furigana** to merge the reading into the main line for a cleaner look.
- **Configurable Ratings**: Choose which rating buttons show up (Again, Hard, Good, Easy) and customize how many options appear. The mapping to Anki’s 1‑indexed ease values is handled automatically.
- **Audio Playback**: Two audio fields are supported:
  - `ankiAudioField` – the default field used by the old UI.
  - `touchBarAudioField` – a new field used when tapping the answer text on the Touch Bar (or the dedicated play/stop button). This lets you keep the answer‑side audio separate from the question side.
- **Layout Toggle**: A new toggle (exposed in the widget settings) moves the **Rating / Audio / Reveal** controls to the left side of the Touch Bar while keeping the **Sync** button on the right. The centre always displays the static "Anki" label.
- **Remaining Counts**: Optionally show the numbers of New, Learning, and Review cards next to the Sync button (or on the left when the toggle is active).

### 📋 JSON Configuration Highlights

Below is an excerpt of the JSON preset for the **Anki Review** widget (the full file lives in `preset/preset.json`).

```json
{
  "type": "Anki Review",
  "title": "Anki Review",
  "iconName": "rectangle.stack.fill",
  "backgroundColorHex": "#1E1E24",
  "textColorHex": "#FFFFFF",
  "fontSize": 18,
  "ankiTextMaxWidth": 580,
  "ankiDeckName": "Kaishi 1.5k",
  "ankiQuestionField": "Word,Sentence",
  "ankiAnswerField": "Word Reading,Sentence Furigana",
  "ankiAudioField": "Sentence Audio",
  "touchBarAudioField": "Word Audio",
  "ankiCombineFurigana": true,
  "ankiShowAgain": true,
  "ankiShowHard": false,
  "ankiShowGood": true,
  "ankiShowEasy": false,
  "ankiShowRemainingCounts": true,
  "ankiFuriganaFontSize": 16,
  "ankiFuriganaVerticalOffset": -3,
  "ankiBoldColorHex": "#00F900"
}
```

**Important fields to note**:
- `touchBarAudioField`: The field that will be played when the user taps the answer text on the Touch Bar.
- `ankiCombineFurigana`: When `true`, furigana is merged into the main line; when `false`, it is displayed as a small ruby annotation above the kanji.
- `ankiShowRemainingCounts`: Shows the New/Learning/Review counters.
- Layout toggle is stored in `UserDefaults` under the key `AnkiTouchBar.isMediaOnLeft` and can be switched from the widget configuration UI.

### 🚀 Getting Started with Anki

1. Open Anki on your Mac.
2. Install the **AnkiConnect** add‑on (Code: `2055492159`).
3. Ensure AnkiConnect is running on the default port `8765`.
4. Add the **Anki Review** widget via TouchBarCraft Settings and select your deck.
5. Use the **Layout Toggle** in the widget settings to choose the preferred button arrangement.

## ⚙️ How to Build and Run

### Prerequisites
- macOS 14.0 or newer
- Swift 5.9+ / Xcode Command Line Tools

### 1. Build the App
Run the provided build script to compile the application and package it into a standard macOS `.app` bundle:

```bash
chmod +x build_app.sh
./build_app.sh
```

### 2. Run the App
- Double‑click **`TouchBarCraft.app`** in Finder to launch it.
- **Recommendation**: Move `TouchBarCraft.app` to your `/Applications` directory to allow macOS to register it for the *Launch at Login* service properly.
